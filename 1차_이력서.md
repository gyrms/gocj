## 자기소개

7년간 호텔·리조트 도메인의 B2C 서비스와 다양한 외부 시스템 연동을 경험한 백엔드 개발자입니다.

야놀자 제휴 약 500개 호텔/모텔을 대상으로 한 온라인 예약 플랫폼을 2인 팀에서 주도적으로 설계하고 개발하며, 소셜 로그인·PG사 결제·SMS 알림·Redis 세션 기반 멀티 도메인 인증 등 실서비스 수준의 B2C 시스템을 구축한 경험이 있습니다.

단순히 주어진 기능을 구현하는 것에 그치지 않고, 레거시 코드 리팩토링(iBatis → MyBatis, 팩토리 메소드 패턴 도입), Spring Framework 버전 업그레이드, AWS 인프라 전환 등 기술 부채 해소와 시스템 개선에도 적극적으로 참여해왔습니다.

앞으로는 대용량 트래픽을 다루는 이커머스 플랫폼에서 더 넓은 도메인과 기술적 깊이를 쌓고 싶습니다.

## 자기소개서

### 1. 지원 동기

야놀자 제휴 예약 플랫폼을 개발하면서 B2C 서비스가 사용자에게 직접 닿는 순간의 무게감을 몸소 경험했습니다. 소셜 로그인부터 결제, 예약까지 이어지는 사용자 여정을 설계하고 구현하며, 더 많은 사용자가 이용하는 대규모 커머스 플랫폼에서 일하고 싶다는 목표가 생겼습니다.

CJ올리브영은 온·오프라인을 아우르는 O2O 커머스 플랫폼으로, 대용량 트래픽과 복잡한 도메인을 기술로 해결하는 경험을 쌓을 수 있는 곳이라고 생각합니다. 제가 가진 B2C 서비스 개발 경험과 외부 시스템 연동 역량을 커머스 플랫폼에서 더욱 발전시키고 싶어 지원하게 됐습니다.

---

### 2. 기술적 역량 및 경험

**Spring Boot / JPA 기반 B2C 서비스 개발**
야놀자 제휴 예약 플랫폼(YPT)에서 Spring Boot와 JPA를 활용해 예약·사용자 도메인을 설계하고 구현했습니다. 2인 팀에서 대부분의 기능을 주도적으로 개발하며, 빠른 의사결정과 실행 능력을 길렀습니다.

**Redis를 활용한 세션 관리**
서로 다른 도메인을 사용하는 웹 서비스 2개 간의 통합 로그인 문제를 Redis 세션 중앙화로 해결했습니다. 쿠키의 도메인 제약을 극복하고 사용자가 두 서비스 모두에서 끊김 없이 로그인 상태를 유지할 수 있도록 구현했습니다.

**다양한 외부 시스템 연동 경험**
카카오·네이버 OAuth2 소셜 로그인, PG사 결제 연동, SMS 알림 서비스, Opera PMS SOAP 통신 등 다양한 외부 시스템과의 연동 경험을 보유하고 있습니다. 인터페이스 설계부터 예외 처리까지 실무에서 직접 다뤘습니다.

---

### 3. 문제 해결 경험

**[상황]** 야놀자 예약 플랫폼 개발 중, 서비스 특성상 도메인이 다른 두 개의 웹 페이지(예약 사이트 / 관리자 페이지)에서 하나의 로그인 세션을 공유해야 하는 요구사항이 있었습니다.

**[과제]** 브라우저 쿠키는 도메인 간 공유가 불가능하여 기존 방식으로는 해결할 수 없었습니다. 사용자가 두 도메인 모두에서 별도 로그인 없이 인증 상태를 유지해야 했습니다.

**[행동]** Redis를 세션 저장소로 사용하고, 두 웹 서비스가 동일한 Redis 인스턴스를 바라보도록 구성했습니다. 세션 키를 공유하는 방식으로 도메인 간 인증 상태를 동기화하여 문제를 해결했습니다.

**[결과]** 사용자는 예약 사이트와 관리자 페이지 중 어느 한 곳에서만 로그인하면 두 서비스 모두에서 인증 상태가 유지되는 통합 로그인 경험을 제공할 수 있었습니다.

---

### 4. 성장 의지

현재 Kotlin과 QueryDSL을 학습하며 올리브영의 기술 스택에 맞춰 준비 중입니다. 또한 레거시 시스템을 직접 개선하고 마이그레이션한 경험(iBatis → MyBatis, Spring 업그레이드, AWS 전환)을 바탕으로, 새로운 환경에서도 빠르게 적응하고 기여할 수 있다고 자신합니다. 앞으로도 기술 블로그 작성과 오픈소스 기여를 통해 지속적으로 성장해 나갈 계획입니다.
